## CONVENTIONS
- **Vue Vine** (`.vine.ts`) instead of `.vue` SFCs ‚Äî components are TS functions with embedded templates
- **File-based routing** via vue-router/vite ‚Äî pages in `pages/` dirs auto-register as routes
- **WXT storage + Vue Query** for state ‚Äî NOT Pinia. Uses `storage.defineItem` + `useQuery`/`useMutation`
- **defineVibe** (vue-vine) for lightweight entrypoint-scoped stores (sidepanel only)
- **PrimeVue + TailwindCSS v4** for UI ‚Äî use PrimeVue components, Tailwind for layout
- **neverthrow** for error handling ‚Äî use `Result`/`Err`/`Ok`, not try/catch
- **Zod v4** for validation ‚Äî schemas in agent/tool definitions

## ANTI-PATTERNS (THIS PROJECT)
- **No ESLint/Prettier** ‚Äî no linting or formatting configured. Match surrounding code style.
- **No test framework** ‚Äî zero tests. Do not add test files unless explicitly requested.
- **env.json contains secrets** ‚Äî API keys in plaintext. Do not reference these in code comments.
- **typed-router.d.ts is auto-generated** ‚Äî never edit manually.
- **NO Pinia** ‚Äî do NOT use `defineStore`. Use `storage.defineItem` + Vue Query or `defineVibe`.

## UNIQUE STYLES
- Two separate Vue apps (popup + sidepanel) with duplicated boot code ‚Äî no shared `createApp` factory
- Background script uses LangGraph agent loop with middleware (trim/sum messages) + streaming via `execAgentLoop`
- Browser bookmarks as knowledge storage layer (not a database)
- Content script is a placeholder (console.log only, matches `*://*.google.com/*`)
- Model adapter supports OpenAI, Anthropic, Google providers via `createModelAdapter`
- Agent loop supports abort control, tool call streaming, reasoning content streaming
